Detailed Description

Application (transaction flow) capabilities (see EMV_CT_APPLIDATA_STRUCT::App_FlowCap)

Macro Definition Documentation

◆ APP_FLOW_CASHBACK_TRANS_TYPE_00

#define APP_FLOW_CASHBACK_TRANS_TYPE_00   0x01

B5b1: For cashback transaction (amount other != 0) use transaction type '00' as required by Visa.

EMV_CT_APPLIDATA_TYPE.TxnTypesCashback will be adjusted automatically to include '00' for CVM processing.
For L2 kernel version <= 7.0.3 there is a work-around to temporarily set back to given transaction type (usually '09', must not indicate sale or cash) during AUC checks.

◆ APP_FLOW_VISA_QUASI_CASH

#define APP_FLOW_VISA_QUASI_CASH   0x80

B4b8: Convert transaction type from '01' to '11' indicating VISA Quasi Cash. Use in conjunction with EMV_CT_APPLIDATA_TYPE.TxnTypesCash if AUC checks shall be applied, that is both values '01' and '11' should be set.

◆ EMV_CT_FALLBACK_ON_INCONS_TRACK2

#define EMV_CT_FALLBACK_ON_INCONS_TRACK2   0x10

B3b5: Valid only in case of activated EMV_CT_CHECK_INCONS_TRACK2_PAN.
If activated: Check will be done before Cardholder Verification. In case of fail EMV ADK will process fallback handling (may result in EMV_ADK_FALLBACK). E.g. required for German dcpos 2.4, may be used for "M/Chip Functional Architecture", AMEX ED-031 in test plan ver. 4.2.

◆ TRANSACTION_TYPE_30_FOR_PREAUTH

#define TRANSACTION_TYPE_30_FOR_PREAUTH   0x80

B2b8: Amex Pre-Authorization with transaction type 30.


Use EMV_ADK_TRAN_TYPE_AMEX_PREAUTH for pre-authorization resp. reservation transactions
The modified transaction type is used towards card and put out in EMV_CT_TRANSRES_STRUCT::T_9C_TransType.

Note
Pre-Authorization and reservation transactions are equivalent. They are initiated with EMV_ADK_TRAN_TYPE_INIT_RESERVATION_A or EMV_ADK_TRAN_TYPE_INIT_RESERVATION_B.

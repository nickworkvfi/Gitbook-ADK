ADK-SDI-API Programmers Guide

ADK-SDI-CLIENT provides a C++ client for the SDI server with an easy to use interface:

  • use function parameters instead of TLV buffers
  • provide dedicated data types for simple use instead of using byte buffers
  • provide a single error code instead of error codes distributed over several places

Usage

SDI Client

SDIClient is the main class for accessing SDI server. Before commands can be invoked the connection needs to be connected using SDIClient::connect, either using a host name and port or by providing an already connected IPC link.

The connection state can be checked using SDIClient::connected.

SDIClient supports several protocol variants, which can be switched using SDIClient::setProtocol. SDIClient::PROTOCOL_B is for the main SDI server access, which only one instance may used and SDIClient::PROTOCOL_C is for concurrent but only limited access (e.g. printing or PIN entry).

Most functions return an error code, some return an error code or status code. All error codes are <0, EMV status codes are >0. No error is indicated by return of OK (0).

Parameter data

On the low level, parameters are sent to SDI server using TLV data. Normally SDIClient already handles those details, so that parameters can be just passed as function parameters. In case direct access is required the class TLV is provided to ease handling of TLV data.

In case of EMV configuration data and EMV transaction data, two classes are provided to simplify access to EMV parameter data: EMVConf and EMVTxn.

For accessing BCD values, fixed sized arrays and bit masks, the helper classes BCD, Array and BitMask are provided.

For EMV bit masks the following classes (based on BitMask) are provided:

  • EMVAppFlowCapabilities
  • EMVKernelAppFlowCapabilities
  • EMVFlowOptions
  • EMVChecksumOptions
  • EMVTransactionOptions
  • EMVTransactionSteps
  • EMVDomesticOptions

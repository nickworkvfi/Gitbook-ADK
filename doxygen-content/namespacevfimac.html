◆ sysLaunchCPAppByTrigger()

MAC_API macError vfimac::sysLaunchCPAppByTrigger ( const std::string &  appid,
const std::string &  trigger_id,
const vfiipc::JSObject &  param,
unsigned int  flags 
)

Will execute all CP applications, which are registered for certain trigger identifier

Parameters
[in]appidapplication id from manifest file (can be "*" for broadcast mode or "1" for unicast mode)
[in]trigger_idtrigger identifier
[in]paramparameters will be passed to CP application in cp_notifyParam field
[in]flagsflags passed to CP app in cp_notifyFlags field
Returns
Returns an macError value enum
MAC_ERR_OK success
MAC_ERR_IPC communication error
MAC_ERR_PARAMETER wrong parameter passed
MAC_ERR_NOT_CP_APP no CP application to launch
MAC_ERR_CPR_DISABLED CPR is disabled
Note
if appid is equal to "*", this will invoke broadcast mode. In this case all CP applications with matching trigger identifier will be launched one by one. Launching sequence happens according to CP application priorities.
if appid is equal to "1", this will invoke unicast mode. In this case only one CP applications with highest priority and matching trigger identifier will be launched.
in all other cases only one application with matching appid will be launched.
Examples
mac_demo1.cpp.

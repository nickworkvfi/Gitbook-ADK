Detailed Description

Used in EMV_CTLS_CALLBACK_FnT.

Used in EMV_CT_CALLBACK_FnT.

The tag determines which callback function is to be used. Data is enclosed inside this contructed tag. Detailed Description uses following convention:
INPUT: This is what Application gets from the Framework
OUTPUT: This is what Application is supposed to send back

◆ TAG_BF04_CBK_REDUCE_CAND

#define TAG_BF04_CBK_REDUCE_CAND   0xBF04

Application Candidate List Reduction/Modification.

Called depending on MS_RETURN_CALLBACKS and EMV_CT_SELOP_CBCK_APPLI_SEL
The application gets the candidate list and has the option to modify it.
It may select which candidate to choose, or reduce the candidate list, or re-order it.
So a selection by the cardholder can be implemented. Or a selection by local rules.
INPUT:
TAG_DF74_CBK_SELECTOR: Who has to select the application (0=merchant or 1=customer)
Following parameters are repeated for each candidate:
TAG_50_APP_LABEL: Application label, the chosen application label according EMVCo rules and application configuration,
TAG_DF04_AID: AID
TAG_DF60_VELOCITY_ORIG_IDX: Original candidate index
TAG_5F2D_LANGUAGE: Language preference
TAG_DFD00B_APPNAME_UFT8: Application label converted to UTF-8 unless switched off by transaction flow option EMV_CT_SELOP_NO_UTF8
TAG_9F11_ISS_CODE_TABLE_ID: Code page index part of ISO-8859, length 0 if not present or discarded, absent if no candidate requires
OUTPUT:
TAG_DF75_CBK_APP_NO: Index of the selected application (1-based, 0=cancel, >0xE0: 0xED=fallback, else abort).
A (resorted) list of candidates can be given back by including several indices in DF75.

Note
Due to backward compatibility the candidate-wise input data (all but selector) are put the order listed here each as sequence of all candidates, e.g. DF74, 50, 50, 50, DF04, DF04, DF04 and so on.

◆ TAG_BF06_CBK_LOCAL_CHECKS

#define TAG_BF06_CBK_LOCAL_CHECKS   0xBF06

Callback for local checks after reading the PAN.

Called depending on MS_RETURN_CALLBACKS and EMV_CT_TRXOP_LOCAL_CHCK_CALLBACK
After step "READ RECORD" the available card data is given to the application.
Upon this information the application can modify certain transaction parameters.
INPUT:
TAG_9F42_APP_CURRENCY_CODE: Transaction currency
TAG_5A_APP_PAN: Primary Account Number
TAG_9F02_NUM_AMOUNT_AUTH: Transaction amount
TAG_9F1B_TRM_FLOOR_LIMIT: Floor limit
TAG_5F28_ISS_COUNTRY_CODE: Issuer country code
TAG_5F24_APP_EXP_DATE: Expiry Date of the card
TAG_DF5B_DCC_PROHIBIT: Info if Dynamic Currency Conversion is forbidden (1=forbidden, 0=allowed), that's the case if currency or amount was already sent to ICC in PDOL
TAG_DF5C_DOM_INFO: Special information, Domestic callback information
TAG_DF21_TAC_DENIAL, TAG_DF22_TAC_ONLINE, TAG_DF23_TAC_DEFAULT: Terminal action codes
TAG_57_TRACK2_EQUIVALENT: Track2 equivalent data
TAG_50_APP_LABEL: Application label
TAG_8E_CVM_LIST: List of Cardholder Verification Methods
OUTPUT (all optional):
TAG_9F02_NUM_AMOUNT_AUTH: Modified transaction amount
TAG_9F1B_TRM_FLOOR_LIMIT: Modified floor limit
TAG_DF76_CBK_MANIPUL_TRX: Modify action analysis, possible values are EMV_ADK_ARQC, EMV_ADK_AAC, EMV_ADK_ABORT
TAG_DF5D_DOM_OPTION: Option for transaction proceeding, Domestic callback options
TAG_DF21_TAC_DENIAL, TAG_DF22_TAC_ONLINE, TAG_DF23_TAC_DEFAULT: Modified terminal action codes
TAG_50_APP_LABEL: Modified application label

◆ TAG_BF0C_CTLS_CBK_APP_SELECTED

#define TAG_BF0C_CTLS_CBK_APP_SELECTED   0xBF0C

Signals that application is selected.

Used Kernel can be changed, invoked depending on CLTRXOP_APPSELECTED_CALLBACK.

INPUT: AID (tag TAG_DF04_AID)
INPUT: application label (tag TAG_50_APP_LABEL)
INPUT: application origin index of candidate list (tag TAG_DF60_VELOCITY_ORIG_IDX)
INPUT: application language preferences of candidate list (tag TAG_5F2D_LANGUAGE)
INPUT: application kernel ID (tag TAG_DFD003_Kernel_ID)
INPUT: Processing Result Bitmap of Entry Point (tag TAG_DF63_VELOCITY_EP_PRB), contains flag for 9F2A presence (see emv_cl.h from Velocity Entry Point)
INPUT: FCI Issuer Discretionary Data from Final Select response (tag TAG_BF0C_FCI_ISS_DISCR)
OUTPUT: 3-byte kernel ID to be used for the transaction (tag TAG_DF7E_KERNEL_TO_USE), e.g. 0x020000 for PayPass, use 0xFF0000 to abort the transaction, use 0xFE0000 to silently abort, use 0x000000 to force fallback to contact interface

Note
For restart without LED flickering activate transaction flow option CLTRXOP_REPEAT_L1

◆ TAG_BF12_CBK_MODIFY_CAND

#define TAG_BF12_CBK_MODIFY_CAND   0xBF12

Modify candidate list.

Invoked depending on CLTRXOP_CANDLIST_CALLBACK.
EMV_CTLS_GetCandidateData() may be called inside this callback to fetch additional candidate data.

INPUT: All AIDs of candidate list (each in a single tag TAG_DF04_AID)
INPUT: All application labels of candidate list (each in a single tag TAG_50_APP_LABEL)
INPUT: All application origin indices of candidate list (each in a single tag TAG_DF60_VELOCITY_ORIG_IDX)
INPUT: All application language preferences of candidate list (each in a single tag TAG_5F2D_LANGUAGE)
INPUT: All application kernel IDs of candidate list (each in a single tag TAG_DFD003_Kernel_ID)
INPUT: All combination Processing Result Bitmap of Entry Point (tag TAG_DF63_VELOCITY_EP_PRB), contains flag for 9F2A presence (see emv_cl.h from Velocity Entry Point)
OUTPUT:

  • TAG_DF7F_CBK_COMM_ERR
  • Reduced list as a list of App Index (TAG_DF75_CBK_APP_NO), resorting only supported for velocity kernel
    A single index > 'E0' causes abort: 'FE' results in silent EMV_ADK_ABORT, others result in EMV_ADK_TXN_EMPTY_LIST with silence depending on CLTRXOP_SILENT_ON_EMPTY_LIST
    Note
    For restart without LED flickering activate transaction flow option CLTRXOP_REPEAT_L1
  • PostProcessingOption, skip Post Processing if TRUE (TAG_DF76_CBK_APP_POSTPROC)
  • Transaction amount update (TAG_9F02_NUM_AMOUNT_AUTH). An amount update will restart the Entry Point pre-processing and combination selection. The Select PPSE APDU exchange is replayed (no command is sent to card). A second Candidate List Callback will be issued because the Entry Point might have other processing results. Amount updates in this second callback are not foreseen and ignored.
  • TAG_DF6C_CBK_REPEAT: Decide if ADK should repeat the callback call or not.
